## 1. 서버와 클라이언트

| 구분 | 설명 |
|------|------|
| **클라이언트** | 서버로 요청을 보내는 프로그램 |

## 2. 데이터베이스

**데이터베이스**란 데이터를 한 곳에 모아두고 여러 사람이 공유·활용할 목적으로 관리하는 데이터 저장소입니다.

### RDB (Relational Database, 관계형 데이터베이스)

- 데이터를 **행(Row)과 열(Column)** 로 이루어진 테이블 형태로 관리
- **기본키(Primary Key)** 를 사용해 각 행을 고유하게 식별

### SQL (Structured Query Language)

- 관계형 데이터베이스에서 **데이터를 조회·조작**하기 위해 사용하는 언어

### NoSQL (Not Only SQL)

- 관계형 모델에 얽매이지 않는 유연한 데이터베이스 방식
- 대용량 데이터 처리에 유리하며, 확장 방식에 따라 두 가지로 구분

| 확장 방식 | 설명 |
|-----------|------|
| **스케일 업 (Scale Up)** | 기존 서버의 성능(CPU, RAM 등)을 높여 처리 능력을 향상 |
| **스케일 아웃 (Scale Out)** | 서버(머신)를 여러 대로 늘려 부하를 분산 |

---

## 3. IP와 포트

- **IP (Internet Protocol)** : 인터넷에서 컴퓨터나 기기들이 서로를 식별하고 통신하기 위한 주소
  - 비유: 백화점 건물 주소
- **포트 (Port)** : 하나의 서버에서 운용 중인 여러 서비스를 구분하기 위한 번호
  - 비유: 백화점 내 개별 매장 번호

---

## 4. 라이브러리 (Library)

- 애플리케이션 개발에 필요한 **클래스, 함수 등을 모아놓은 코드의 집합**
- 특정 기능이 필요할 때 가져다 사용하는 방식으로 개발 생산성을 높임

> **예시 — 로그 라이브러리**
> - 로그 생성 기능
> - 다양한 로그 포매팅 지원
> - 로그 레벨(DEBUG, INFO, WARN, ERROR 등) 설정 기능

---

## 5. 프레임워크 (Framework)

- 애플리케이션의 **전체적인 구조(뼈대)를 잡아주는 소프트웨어 개발 환경**
- 개발자가 비즈니스 로직에 집중할 수 있도록 공통 기능을 미리 제공
- 라이브러리와의 차이: 프레임워크가 전체 흐름을 제어하고, 개발자의 코드를 호출함 *(제어의 역전, IoC)*

---

## 6. 자바 애너테이션 (Java Annotation)

- 자바 코드에 추가하는 **메타데이터 표식**
- 컴파일러나 런타임에 특정 동작을 지시하거나 정보를 제공하는 역할

### 대표적인 기본 애너테이션

| 애너테이션 | 설명 |
|------------|------|
| `@Override` | 부모 클래스(또는 인터페이스)의 메서드를 오버라이드했음을 명시 |
| `@Deprecated` | 더 이상 사용을 권장하지 않는 요소임을 표시 |
| `@SuppressWarnings` | 컴파일러 경고를 무시하도록 지시 |
