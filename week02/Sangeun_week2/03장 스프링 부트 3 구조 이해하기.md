## 1. 계층 구조 (Layered Architecture)

```
클라이언트 요청
      ↓
┌─────────────────────────┐
│   프레젠테이션 계층       │  HTTP 요청 수신 → 비즈니스 계층으로 전달
├─────────────────────────┤
│   비즈니스 계층           │  서비스 로직 처리 (핵심 비즈니스 규칙)
├─────────────────────────┤
│   퍼시스턴트 계층         │  데이터베이스 관련 로직 처리
└─────────────────────────┘
      ↕
  데이터베이스
```

| 계층 | 역할 |
|------|------|
| **프레젠테이션 계층** | HTTP 요청을 받고 비즈니스 계층으로 전송 |
| **비즈니스 계층** | 서비스를 만들기 위한 모든 비즈니스 로직 처리 |
| **퍼시스턴트 계층** | 모든 데이터베이스 관련 로직 처리 |

## 2. 프로젝트 디렉터리 구조

```
project/
├── src/
│   ├── main/       ← 실제 소스 코드 및 리소스 파일
│   └── test/       ← 테스트용 코드 및 리소스 파일
├── build.gradle    ← 빌드 설정 파일
└── settings.gradle ← 프로젝트 정보 설정 파일
```

### 주요 파일 설명

| 파일 / 디렉터리 | 설명 |
|----------------|------|
| `src/main` | 실제 코드를 작성하는 공간. 프로젝트 실행에 필요한 소스 코드와 리소스 파일 위치 |
| `src/test` | 프로젝트 소스 코드를 테스트할 목적의 코드와 리소스 파일 위치 |
| `build.gradle` | 빌드 설정 파일. 의존성(Dependency) 및 플러그인 설정 등 빌드에 필요한 설정 작성 |
| `settings.gradle` | 빌드할 프로젝트의 정보를 설정하는 파일 |
