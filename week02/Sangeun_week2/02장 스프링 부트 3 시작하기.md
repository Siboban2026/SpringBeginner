## 1. 스프링 핵심 개념

| 개념 | 설명 |
|------|------|
| **IoC** (Inversion of Control, 제어의 역전) | 객체의 생성과 관리를 개발자가 직접 하는 것이 아니라 프레임워크가 대신하는 것 |
| **DI** (Dependency Injection, 의존성 주입) | 외부에서 객체를 주입받아 사용하는 것 |
| **AOP** (Aspect-Oriented Programming, 관점 지향 프로그래밍) | 핵심 관점과 부가 관점을 분리하여 개발하는 것 |
| **PSA** (Portable Service Abstraction, 이식 가능한 서비스 추상화) | 어떤 기술을 사용하더라도 일관된 방식으로 처리하도록 추상화하는 것 |

## 3. 자바 17 주요 문법

### 텍스트 블록 (Text Block)

여러 줄의 문자열을 `"""` 로 감싸서 작성하는 방식으로, `\n` 없이도 줄바꿈이 적용

```java
String textBlock = """
        {
            "id": 1,
            "name": "juice"
        }
        """;
```

`formatted()` 메서드를 함께 사용하면 값을 동적으로 삽입 가능

```java
String textBlock17 = """
        {
            "id": %d,
            "name": "%s"
        }
        """.formatted(2, "juice");
```

### 레코드 (Record)

데이터 전달을 목적으로 하는 객체를 간편하게 만들기 위한 기능
파라미터는 자동으로 `private final` 로 정의되며, getter가 자동으로 생성

```java
record Item(String name, int price) {
    // 별도 boilerplate 코드 불필요
}

Item juice = new Item("juice", 3000);
juice.price(); // 3000
```

## 4. @SpringBootApplication

스프링 부트 애플리케이션의 진입점으로, `main()` 메서드가 위치한 클래스에 선언
아래 세 가지 애너테이션을 합친 복합 애너테이션

```
@SpringBootApplication
├── @SpringBootConfiguration  → 스프링 부트 관련 설정 클래스임을 명시
├── @ComponentScan            → 사용자가 등록한 빈을 탐색·등록
└── @EnableAutoConfiguration  → 스프링 부트 자동 구성 활성화
```

### @Component 계열 애너테이션

`@Controller`, `@RestController`, `@Configuration`, `@Repository`, `@Service` 는 모두 `@Component` 를 포함하고 있어 **@ComponentScan** 에 의해 빈으로 자동 등록

| 애너테이션 | 설명 |
|------------|------|
| `@Configuration` | 설정 파일 등록 |
| `@Repository` | ORM 매핑 |
| `@Controller`, `@RestController` | 라우터 (HTTP 요청 처리) |
| `@Service` | 비즈니스 로직 |
