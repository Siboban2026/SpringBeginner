## 1. 테스트 코드

테스트 코드 작성을 통해 코드의 기능이 정상적으로 동작하는지 검증

### Given / When / Then 구조

| 단계 | 설명 |
|------|------|
| **Given** | 테스트 준비 단계 |
| **When** | 테스트 실행 단계 |
| **Then** | 테스트 결과 검증 단계 |

```java
@DisplayName("새로운 메뉴를 저장한다.")
@Test
public void saveMenuTest() {
    // 1. given : 메뉴를 저장하기 위한 준비 과정
    final String name = "아메리카노";
    final int price = 2000;
    final Menu americano = new Menu(name, price);

    // 2. when : 실제로 메뉴를 저장
    final long savedId = menuService.save(americano);

    // 3. then : 메뉴가 잘 추가되었는지 검증
    final Menu savedMenu = menuService.findById(savedId).get();
    assertThat(savedMenu.getName()).isEqualTo(name);
    assertThat(savedMenu.getPrice()).isEqualTo(price);
}
```

## 2. JUnit

자바 단위 테스트 프레임워크로, **메서드 단위**의 기능을 검증

### 특징

- 테스트를 구분하는 애너테이션 제공
- `@Test` 사용 시 매 실행마다 새로운 인스턴스 생성
- 예상 결과 검증을 위한 assertion 메서드 제공
- 간단한 사용 방식으로 빠른 테스트 코드 작성 가능
- 자동 실행 및 결과 확인 기능 제공

---

## 3. JUnit 생명주기

```
@BeforeAll → @BeforeEach → @Test → @AfterEach → @AfterAll
```

### 애너테이션 정리

| 애너테이션 | 실행 시점 | 목적 | static |
|------------|-----------|------|--------|
| `@BeforeAll` | 전체 테스트 시작 전 1회 | 테스트 환경 초기화, 공통 자원 설정 | 필요 |
| `@BeforeEach` | 각 테스트 케이스 실행 전마다 | 객체 초기화, 테스트 데이터 준비 | 불필요 |
| `@AfterEach` | 각 테스트 케이스 종료 후마다 | 테스트 데이터 정리 | 불필요 |
| `@AfterAll` | 전체 테스트 종료 후 1회 | 자원 해제, 종료 처리 | 필요 |

### 실행 예시 코드

```java
import org.junit.jupiter.api.*;

public class JUnitCycleTest {

    @BeforeAll
    static void beforeAll() {
        System.out.println("@BeforeAll");
    }

    @BeforeEach
    public void beforeEach() {
        System.out.println("@BeforeEach");
    }

    @Test
    public void test1() { System.out.println("test1"); }

    @Test
    public void test2() { System.out.println("test2"); }

    @Test
    public void test3() { System.out.println("test3"); }

    @AfterEach
    public void afterEach() {
        System.out.println("@AfterEach");
    }

    @AfterAll
    static void afterAll() {
        System.out.println("@AfterAll");
    }
}
```

## 4. AssertJ

JUnit과 함께 사용하는 검증 라이브러리로, 검증문의 **가독성을 향상**

```java
// JUnit 방식
Assertions.assertEquals(sum, a + b);

// AssertJ 방식 (더 직관적)
assertThat(a + b).isEqualTo(sum);
```

### 주요 메서드

| 메서드 | 설명 |
|--------|------|
| `isEqualTo(A)` | 동일 값 검증 |
| `isNotEqualTo(A)` | 값 불일치 검증 |
| `contains(A)` | 포함 여부 검증 |
| `doesNotContain(A)` | 미포함 검증 |
| `startsWith(A)` | 접두사 검증 |
| `endsWith(A)` | 접미사 검증 |
| `isEmpty()` | 빈 값 여부 검증 |
| `isNotEmpty()` | 비어있지 않음 검증 |
| `isPositive()` | 양수 여부 검증 |
| `isNegative()` | 음수 여부 검증 |
| `isGreaterThan(1)` | 값 초과 비교 |
| `isLessThan(1)` | 값 미만 비교 |

## 6. HTTP 응답 코드 검증

### 주요 응답 코드

| 코드 | 의미 |
|------|------|
| `200 OK` | 정상 응답 |
| `201 Created` | 생성 성공 |
| `400 Bad Request` | 클라이언트 요청 오류 |
| `403 Forbidden` | 접근 금지 |
| `404 Not Found` | 리소스 없음 |
| `500 Internal Server Error` | 서버 오류 |

### 검증 메서드

| 메서드 | 검증 코드 |
|--------|-----------|
| `isOk()` | 200 |
| `isCreated()` | 201 |
| `isBadRequest()` | 400 |
| `isForbidden()` | 403 |
| `isNotFound()` | 404 |
| `is4xxClientError()` | 400번대 전체 |
| `isInternalServerError()` | 500 |
| `is5xxServerError()` | 500번대 전체 |

## 7. 실습 문제

### 문제 01 — GET 테스트

**조건**
- `GET /quiz?code=1` → 응답 코드 `201`, 응답 본문 `Created!`
- `GET /quiz?code=2` → 응답 코드 `400`, 응답 본문 `Bad Request!`

**문제**

```java
@DisplayName("quiz(): GET /quiz?code=1 이면 응답 코드는 201, 응답 본문은 Created!를 리턴한다.")
@Test
public void getQuiz1() throws Exception {
    // 여기에 코드를 작성해주세요
}

@DisplayName("quiz(): GET /quiz?code=2 이면 응답 코드는 400, 응답 본문은 Bad Request!를 리턴한다.")
@Test
public void getQuiz2() throws Exception {
    // 여기에 코드를 작성해주세요
}
```

**정답**

```java
@DisplayName("quiz(): GET /quiz?code=1 이면 응답 코드는 201, 응답 본문은 Created!를 리턴한다.")
@Test
public void getQuiz1() throws Exception {
    // given
    final String url = "/quiz";

    // when
    final ResultActions result = mockMvc.perform(get(url)
            .param("code", "1"));

    // then
    result
        .andExpect(status().isCreated())
        .andExpect(content().string("Created!"));
}

@DisplayName("quiz(): GET /quiz?code=2 이면 응답 코드는 400, 응답 본문은 Bad Request!를 리턴한다.")
@Test
public void getQuiz2() throws Exception {
    // given
    final String url = "/quiz";

    // when
    final ResultActions result = mockMvc.perform(get(url)
            .param("code", "2"));

    // then
    result
        .andExpect(status().isBadRequest())
        .andExpect(content().string("Bad Request!"));
}
```

### 문제 02 — POST 테스트

- `POST /quiz` (body: `code=1`) → 응답 코드 `403`, 응답 본문 `Forbidden!`

**문제**

```java
@DisplayName("quiz(): POST /quiz?code=1 이면 응답 코드는 403, 응답 본문은 Forbidden!를 리턴한다.")
@Test
public void postQuiz1() throws Exception {
    // 여기에 코드를 작성해주세요.
}

@DisplayName("quiz(): POST /quiz?code=13 이면 응답 코드는 200, 응답 본문은 OK!를 리턴한다.")
@Test
public void postQuiz13() throws Exception {
    // 여기에 코드를 작성해주세요.
}
```

**정답**

```java
@DisplayName("quiz(): POST /quiz?code=1 이면 응답 코드는 403, 응답 본문은 Forbidden!를 리턴한다.")

public void postQuiz1() throws Exception {
    // given
    final String url = "/quiz";

            .content(objectMapper.writeValueAsString(new Code(13))));

    // then

        .andExpect(status().isOk())
        .andExpect(content().string("OK!"));
```
