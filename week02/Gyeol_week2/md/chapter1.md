# 1장 자바 백엔드 개발자가 알아두면 좋은 지식

---

## 1.1 서버와 클라이언트

### Client
- 사용자 요청(Request)을 보내는 주체
- 예: 브라우저, 모바일 앱

### Server
- 요청을 처리하고 응답(Response) 반환

### API
- 클라이언트와 서버가 소통하는 규칙

### HTTP
- 웹 통신 프로토콜

---

## 1.2 데이터베이스

### Database(DB)
- 데이터를 저장하는 공간

### DBMS
- 데이터베이스 관리 시스템
- 예: MySQL, PostgreSQL

### SQL
- 데이터 조회/수정 언어

---

## 1.3 아이피와 포트

### IP
- 컴퓨터 주소

### Port
- 프로그램(서비스) 번호
- 예: 8080

### localhost
- 내 컴퓨터 자신
- 127.0.0.1

---

## 1.4 라이브러리와 프레임워크

### Library
- 필요한 기능을 가져다 쓰는 코드 모음
- 내가 호출함

### Framework
- 정해진 흐름 안에서 내가 코드를 넣음
- 프레임워크가 흐름 제어

### 차이
- Library → 내가 부름
- Framework → 나를 부름

---

## 1.5 백엔드 개발자의 업무

### CI (Continuous Integration)
- 지속적 통합
- 저장소에 코드 변경사항을 자주 병합하고 자동 테스트/빌드 수행

### CD (Continuous Deployment/Delivery)
- 지속적 배포
- 변경된 코드를 운영 환경까지 자동 반영

---

## 1.6 백엔드 프로그래밍 언어

### Java
- 객체지향 언어
- Spring에서 많이 사용

### JVM
- 자바 실행 환경

### JDK
- 자바 개발 도구 모음

### JRE
- 자바 실행 환경

---

## 1.7 자바 애너테이션

### Annotation
- @ 로 시작하는 메타데이터

### @Override
- 메서드 재정의 표시

### @SpringBootApplication
- 스프링부트 시작 설정

---

## 헷갈리는 개념 메모
| 개념 | 차이 |
|------|------|
| Library vs Framework | 내가 호출 / 프레임워크가 호출 |
| JDK vs JRE vs JVM | 개발도구 / 실행환경 / 가상머신 |
| CI vs CD | 통합 / 배포 |
